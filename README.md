CS-230 Operating Platforms

This repository contains my completed software design document for CS-230 Operating Platforms.

Reflection
Briefly summarize The Gaming Room client and their software requirements.

The client was The Gaming Room. They wanted to take their game, Draw It or Lose It, which was originally only available on Android, and expand it so players could use it on Windows, Mac, Linux, Android, and iOS. The goal was to design a web-based application that could support multiple users, stay secure, and continue to grow as more players joined.

What did you do particularly well in developing this documentation?

I think I did a good job evaluating the different operating platforms and explaining why Linux was the best choice for the server. I also think I did well connecting the client's requirements to the recommendations instead of just talking about technology for the sake of technology.

What about the process of working through a design document did you find helpful when developing the code?

The design document helped me think through the system before writing any code. It made me consider things like storage, memory management, security, and how the application would communicate across different platforms. Having a plan first makes it easier to understand what needs to be built and why.

If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?

If I could go back and revise one section, I would probably spend more time on the operating system architecture and distributed systems sections. As I learned more throughout the course, I realized there were additional details about memory management, networking, and server architecture that I could have explained better.

How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing?

I started by looking at what The Gaming Room was actually trying to accomplish. They wanted more people to be able to play their game without having to create separate applications for every operating system. That led me toward a web-based design with a Linux server and browser-based clients. Understanding the user's needs is important because a design can be technically impressive and still fail if it doesn't solve the client's actual problem.

How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?

My approach was to start with the requirements and work from there. First I looked at what the client needed, then I evaluated different platforms, architectures, storage options, security features, and development tools. In the future I would use the same process because it helps break a large problem into smaller pieces and makes it easier to choose the best solution for the project.
